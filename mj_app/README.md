# 🀄 麻雀成績可視化アプリ

こちらは生成AIを使用せずに作成したものです。  
Streamlit を使って麻雀の対局成績を**グラフ付きで可視化**する Web アプリです。  
CSV ファイルを読み込むだけで、選手ごとのスコア推移・順位分布・月別分析などを一目で確認できます。

---

## 📁 必要なファイル

アプリを動かすには、`app.py` と同じフォルダに以下の **2 つの CSV ファイル** が必要です。

| ファイル名 | 役割 |
|---|---|
| `set_res.csv` | 全体成績（各対局の 1〜4 着の名前・スコアを記録） |
| `res_input.csv` | 個人詳細成績（素点・収支・対戦グループなどを記録） |

### `set_res.csv` の列構成（例）

| 列名 | 内容 |
|---|---|
| `date` | 対局日 |
| `match` | 対局番号 |
| `member` | 対戦グループ名 |
| `rule` | 使用ルール |
| `1st_name` / `1st_score` | 1 着の名前・スコア |
| `2nd_name` / `2nd_score` | 2 着の名前・スコア |
| `3rd_name` / `3rd_score` | 3 着の名前・スコア |
| `4th_name` / `4th_score` | 4 着の名前・スコア |

### `res_input.csv` の列構成（例）

| 列名 | 内容 |
|---|---|
| `date` | 対局日 |
| `match` | 対局番号 |
| `member` | 対戦グループ名 |
| `ForS` | 形式（四麻・三麻など） |
| `rank` | 着順（1〜4） |
| `score` | 素点 |
| `p-m` | ポイント収支（プラスマイナス） |

---

## 🚀 起動方法

```bash
# 必要ライブラリのインストール（初回のみ）
pip install streamlit pandas plotly

# アプリの起動
streamlit run app.py
```

起動後、ブラウザが自動的に開きます（デフォルト: `http://localhost:8501`）。

## 🖥️ アプリの構成

アプリは左のサイドバーで **2 つのページ** を切り替えて使います。

---

### ページ 1：📊 全体成績分析 (All)

`set_res.csv` のデータを使って、**特定の選手の成績を俯瞰**するページです。

**サイドバーのフィルタ**
- **選手を選択**：分析したい選手を選ぶ（デフォルトは "Hamada"）
- **対戦グループ**：特定のグループの対局だけに絞り込む（`All` で全件表示）

**表示される指標（KPI）**

| 指標 | 説明 |
|---|---|
| 対戦数 | フィルタ後の総対局数 |
| 通算スコア | `score_pm` の合計値 |
| 平均順位 | 着順の平均（低いほど良い） |
| トップ率 | 1 着になった割合 |
| 連対率 | 1・2 着になった割合 |

**グラフ**
- 📈 **通算スコア推移**：対局回数を横軸に、累積スコアの折れ線グラフ
- 🍰 **順位分布**：1〜4 着の割合を円グラフで表示（金・銀・銅・グレー）
- 📝 **対局履歴テーブル**：日付・グループ・着順・スコアの一覧（新しい順）

---

### ページ 2：👤 個人詳細分析 (Personal)

`res_input.csv` のデータを使って、**より細かい個人成績を多角的に分析**するページです。

**サイドバーのフィルタ**
- **グループ (member)**：対戦グループで絞り込む
- **形式 (ForS)**：四麻・三麻などの形式で絞り込む
- **期間**：分析対象の日付範囲を指定

**表示される指標（KPI）**

| 指標 | 説明 |
|---|---|
| 対戦数 | フィルタ後の総対局数 |
| 通算収支 (p-m) | ポイント収支の合計 |
| 平均順位 | 着順の平均 |
| 平均素点 | 素点の平均値 |

**グラフ・分析セクション**

1. 📈 **収支推移 (p-m)**：累積ポイント収支の折れ線グラフ
2. 📊 **素点分布**：素点のヒストグラム（20 ビン）
3. 🏢 **グループ別成績比較**（グループ「All」選択時のみ表示）
   - 各グループの通算収支を棒グラフで比較
   - 各グループの平均順位を棒グラフで比較（Y 軸反転で低い順位が上）
   - 詳細データテーブル（展開式）
4. 📅 **月別データ分析**
   - 月別の収支合計（棒グラフ）
   - 月別の着順内訳（積み上げ棒グラフ）
5. 📝 **詳細データ一覧**：全フィールドを含むテーブル（新しい順）

## ⚙️ 技術スタック

| ライブラリ | 用途 |
|---|---|
| [Streamlit](https://streamlit.io/) | Web UI の構築・ページルーティング |
| [Pandas](https://pandas.pydata.org/) | CSV 読み込み・データ加工・集計 |
| [Plotly Express](https://plotly.com/python/plotly-express/) | インタラクティブなグラフ描画 |

### データ読み込みの仕組み

- `@st.cache_data` デコレータにより、CSV は**初回のみ読み込み**、以降はキャッシュを利用して高速表示します。
- ファイルが存在しない場合は警告を表示し、エラーにはなりません。
- `set_res.csv` は「横持ち（1 行に 4 人分）」のデータを「縦持ち（1 行に 1 人分）」に変換してから使用します。

---

## 📝 補足・注意事項

- CSV のヘッダー名は上記の通りに合わせてください（大文字・小文字も含む）。
- `date` 列は `YYYY-MM-DD` 形式を推奨します。
- デフォルトの選択選手は `"Hamada"` ですが、該当する名前がない場合はリストの先頭が選ばれます。
